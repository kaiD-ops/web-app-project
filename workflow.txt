Workflow 1(stakeholder view by Ammar Danish)
    Actor: Event Stakeholder

Objective: To publish a new event and allocate a budget for audience gathering.

Step-by-Step Description:

Login/Dashboard: The Stakeholder logs in and lands on the "Organizer Dashboard," which displays "Active Events," "Past Events," and "Create New Event."

Initiate Event: User clicks "Create New Event."

Input Details: The system presents a form requiring: Event Name, Date, Time, Venue Location, Expected Duration, and Description.

Requirement Specification: User inputs the "Required Audience Count" (e.g., 50 students) and "Total Budget" (e.g., 5000 PKR).

System Calculation: The system automatically calculates the "Per Student Payout" (Total Budget / Audience Count) - Platform Fee. The Stakeholder reviews this calculated rate.

Submission: The Stakeholder clicks "Publish."

Status Update: The event status becomes "Open," and it is immediately listed on the Student Dashboard.

Workflow 2(by Nayab Dhanani)
Actor: Student

Objective: To find an event, accept the gig, and receive confirmation.

Step-by-Step Description:

Registration (First Time): Student signs up providing Name, University ERP ID, and Payment Details (Account Number). System validates unique ERP to prevent duplicates.

Browse Events: Student logs in to see a list of "Upcoming Events" showing Time, Location, and Hourly Payout.

Selection: Student clicks an event to view full details (Topic, Organizer, Duration).

Confirmation: Student clicks "Confirm Availability."

Validation: System checks if the event is full. If slots are available, the system reserves a spot for the student.

Notifications:

System schedules an automated generic notification 1 hour prior to the event.

System schedules an urgent notification 10 minutes prior to the event.

Status: The event moves to the "My Scheduled Gigs" tab in the student's dashboard.
